Sections
You can define sections in your markup. These are named placeholders. In other parts of the markup such as page views or partial views you can define tags that serve as content for these sections. Before the rendering of the page finishes all such placeholders get evaluated, and their accumulated contents get rendered in place of the placeholder tag.
Let's say just like in Razor, we want to mark the place for extra scripts in the Masterpage, we're using this code for placeholder: 
<div eve-section="scripts" eve-renderinstead></div>

In the LandingPage here's how we define one script to include in this section. 
  <!-- jQuery -->
<script src="/EVE.Mvc.Samples.Embedded/Assets/LandingPage/js/jquery.js" eve-sectioncontentfor="scripts"></script>

In the Contents partial view here's how to define to include another line: 
<div eve-sectioncontentfor="scripts" eve-renderonlycontent>
    <!-- Bootstrap Core JavaScript -->
    <script src="/EVE.Mvc.Samples.Embedded/Assets/LandingPage/js/bootstrap.min.js"></script>
</div>

Here are the meanings of the relevant attributes:
Attribute Description 
eve-section="scripts"
Defines a named section for the page. 
eve-sectioncontentfor="scripts"
Defines a content for a named section. 
eve-renderonlycontent
Defines that only the inner content of a given tag is considered as a content for a named section.
