Using Embedded Razor views

CAVEAT: This is also done differently elsewhere
While it is possible to use Razor views in the with the embedded file system, there is already an established solution to use Razor views in separate dll-s using RazorGenerator
Defining your embedded Razor View
You can embed a Razor view into your class library project. In order for intellisense and other Razor viewer features to work, you will need to include a web.config file in the project as well. See the RazorGenerator link above for more detail on how to get this working. First, let's set up a Razor view and make it embedded.


Defining your embedded File System and virtual path provider
For MVC to locate your razor view you have to register a virtual path provider that is aware of the views embedded location. You can do this at application startup using the following code: 
HostingEnvironment.RegisterVirtualPathProvider(new EmbeddedPathProvider(fs, reqPath));
By providing the embedded file system and the base embedded path to the EmbeddedPathProvider it will be able to find and retrieve the razor view from the embedded location.
Calling the view with the embedded path
Very simple Controller action code to this view would be: 
return View("/EVE.Mvc.Samples.Embedded/Views/Sample/index.cshtml");
